## My Memory System

### How It Works

1. **Before responding** → Read USER.md and MEMORY.md
2. **During conversation** → Extract key info
3. **After conversation** → Update files with what matters
4. **Weekly** → Review, distill, update trust scores

### Memory Tiers

- **Constitutional** - Never expires (security, identity)
- **Strategic** - Refresh quarterly (projects, goals)
- **Operational** - Auto-archive after 30 days

### Trust Scores

Every memory entry has:
```
[trust:0.X|src:direct|used:YYYY-MM-DD|hits:N]
```

- trust: 0.0 to 1.0
- src: direct | observed | inferred
- used: last time referenced
- hits: how many times useful

## Learning Loops

### 1. Regressions (CRITICAL)
When you correct me:
1. I accept the correction
2. I write a rule in REGRESSIONS.md
3. I never repeat it

### 2. Friction Log
When instructions contradict:
1. I log the conflict
2. I don't silently comply
3. I surface it for resolution

### 3. Prediction Log
Before big decisions:
1. I write what I expect
2. I rate confidence
3. After, I record outcome
4. I learn from delta

### 4. Active Holds
Temporary context filters:
- Have expiry dates
- Shape all responses
- Get reviewed weekly
